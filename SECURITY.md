# Security Policy

## Supported Versions

Security fixes are applied to the current minor release line.

| Version | Supported |
| --- | --- |
| 1.1.x | Yes |
| 1.0.x and earlier | No |

Users should install the latest published patch release before reporting a
problem.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability or for a report that
contains credentials, private data, or an unpublished exploit.

Use GitHub's private vulnerability-reporting form when it is available:

<https://github.com/christopherposadasp-ctrl/StatsRPackage/security/advisories/new>

If that form is unavailable, contact the package maintainer at
<christopher.v.posadas@nps.edu>. Include the affected version, reproduction
steps, impact, and any proposed mitigation. Remove real credentials and
sensitive data from all examples.

The maintainer will acknowledge a complete report, assess whether a supported
release is affected, and publish a fixed patch release rather than moving an
existing tag.
